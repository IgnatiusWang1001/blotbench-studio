### 1.1 修复新快照导入被误判为不支持
- `work/app/src/lib/project.ts`
- 之前 `parseProjectSnapshot(...)` 仍然要求旧格式里的 `lanes` 数组存在，导致已经切到 `lanesByPanelId` / `settingsByPanelId` 的新快照反而可能被拒绝导入

- `work/app/src/lib/project.ts`
- `decodedGray.pixels` 原本可能是 `Uint8Array / Uint16Array`
- 直接 `JSON.stringify(...)` 后再恢复时，不会自然回到 typed array，保存的 TIFF 灰度矩阵实际上不可靠
- 现已改为：
  - 导出时序列化为普通 number 数组
  - 导入时按 `bitDepth` 恢复为 `Uint8Array` 或 `Uint16Array`
- 这意味着：
  - Project JSON 更可信
  - autosave 恢复也更可信
  - TIFF 本地半定量的恢复链路真正闭环

- `work/app/src/lib/project.ts`
- `loadAutosave()` 现在会捕获损坏或旧格式异常
