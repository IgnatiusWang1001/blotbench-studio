## 技术方案概述
这个想法可以实现，而且很适合做成一个纯前端优先、可后续增强算法能力的 Web 工具。MVP 的核心目标不是一次性“完全替代所有图像分析软件”，而是把最割裂的四段流程打通为一条流水线：原图上传 -> 条带/泳道裁剪与校正 -> 灰度半定量 -> 自动排版导出。第一版建议优先支持 Western Blot、Dot Blot 和常规凝胶图像的“半自动分析 + 一键出图”，在浏览器内完成项目保存、参数调整、统计图生成与 SVG/PDF 导出。

## 技术选型
| 类别 | 选择 | 理由 |
|------|------|------|
| 语言 | TypeScript | 前后共享数据结构，前端图像与导出逻辑更易维护 |
| 前端框架 | React + Vite | 适合做强交互裁剪界面，构建快，便于后续部署 GitHub Pages |
| 图像交互 | Canvas/Konva | 适合实现拖拽裁剪框、泳道对齐、可视化蒙版 |
| 图像算法 | OpenCV.js + 自定义灰度积分算法 | 浏览器内即可做阈值、背景扣除、轮廓检测、几何校正 |
| 图表 | SVG 原生生成或 D3 | 便于导出 publication-ready 矢量图 |
| 状态管理 | Zustand | 轻量，适合多步骤工作流状态 |
| 数据导出 | SVG + jsPDF/pdf-lib | 支持高质量矢量导出 |
| 本地存储 | IndexedDB | 保存项目、原图、分析参数，避免用户刷新丢失 |

## 架构设计
整体建议采用纯前端本地优先架构，后续再按需增加服务端。

表现层：
- 上传页：拖拽上传 TIFF/JPG/PNG，多图项目管理
- 分析页：图像裁剪、泳道标注、条带识别、灰度预览
- 出图页：上方拼胶图，下方柱状图/散点图，版式微调

业务层：
- 图像预处理：灰度化、反相、去噪、旋转校正、亮度归一
- ROI 与泳道模型：泳道、条带、背景区、分组、内参映射
- 半定量引擎：积分灰度、背景扣除、内参归一化、组间汇总
- 图形排版引擎：拼图、标注、图例、统计图、导出

数据层：
- 浏览器内文件缓存
- 项目 JSON 序列化
- 导出 SVG/PDF/CSV

## 核心算法设计
### 1. 图像预处理
- TIFF/JPG 统一转为灰度矩阵
- 提供“深色条带/浅色背景”和“浅色条带/深色背景”两种模式
- 支持用户手动反相、Gamma 调整、局部对比度增强
- 默认保留原图，不直接覆盖

### 2. 泳道与条带识别
MVP 不建议追求完全自动识别。更稳妥的方案是“自动初猜 + 人工校正”：
- 用户先框选整个 blot/gel 区域
- 算法按纵向投影或横向投影找泳道分界
- 每条泳道内按强度峰值找条带中心
- 根据用户指定的目标蛋白和内参行，自动生成一组标准 ROI
- 用户可拖动、统一拉伸、批量对齐

这样比 ImageJ 少了大量机械画框，但不会因为全自动失误毁掉整个实验。

### 3. 灰度半定量
对每个 ROI 计算：
- `Integrated Density = sum(pixel_intensity_in_band_area)`
- 局部背景：取条带周围环形区或上下邻近区估计背景
- `Corrected Density = Integrated Density - background_mean * band_area`
- 目标蛋白归一化：`Target / LoadingControl`

Dot Blot 可直接按 spot ROI 做同样积分，不一定需要泳道逻辑。

### 4. 组间统计
应用层支持：
- biological replicate 聚合
- 平均值、SD、SEM
- t-test / one-way ANOVA 的前端计算

但应明确：
- 统计学检验依赖样本设计，不能仅凭图像本身“自动正确”
- 工具应要求用户确认 n、分组、是否配对

## 版式与导出设计
目标不是只导出“图”，而是导出一张可直接进论文或汇报的图版。

建议版式引擎支持：
- 上方：裁剪后的 blot/gel 拼接图
- 每条泳道下方自动显示 sample/group label
- 右侧或底部：目标蛋白、内参名称、分子量标注
- 下方：柱状图，可叠加每个 replicate 的散点
- 显著性横线和 `* / ** / ***`
- 整体导出为 SVG，PDF 由 SVG 转换生成

图形实现上建议全部使用 SVG 生成，而不是截图导出。这样文本、线条和柱状图都保持矢量。

## 关键设计决策
| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前后端架构 | 纯前端优先 | 隐私好、部署简单、无上传等待 |
| 自动识别策略 | 半自动 | 实验图像变异大，完全自动误检风险高 |
| 图像分析库 | OpenCV.js | 浏览器可运行，功能足够做阈值和轮廓分析 |
| 导出格式 | SVG 优先，补充 PDF | 更适合 publication-ready 版式 |
| 统计功能深度 | 基础统计内置，高级统计后置 | 避免第一版把重点从图像流程打散 |

## MVP 范围
第一版建议只做以下闭环：
1. 上传单张或多张 WB/gel 图片
2. 手动框定 blot 区域
3. 自动切分泳道并生成条带 ROI
4. 允许批量对齐与微调
5. 自动算目标蛋白/内参比值
6. 自动生成柱状图 + 散点
7. 导出 SVG/PDF/CSV

先不要在 MVP 里做：
- 云端协作
- AI 语义识别“哪一行是 GAPDH”
- 复杂统计模型
- 完全无人校正的自动识别
- Photoshop 级图像精修

## 风险与科研合规点
- 饱和曝光会让灰度失真，工具应提示用户检查 saturation
- 不同曝光时间的图不能直接混算，需显式标记来源
- 背景扣除方法会显著影响结果，必须可见、可调、可复现
- 论文图版需要保留原始 ROI 和参数记录，方便审稿时回溯
- 工具应导出分析报告 JSON/CSV，记录每个 band 的 ROI、背景和计算公式

## 实现计划
1. 第一步：搭建 React + Vite + TypeScript 工程，完成图像上传与项目状态模型
2. 第二步：完成 blot 区域裁剪、泳道框编辑、批量对齐 UI
3. 第三步：接入 OpenCV.js，完成泳道投影和条带峰值检测
4. 第四步：实现积分灰度、背景扣除、内参归一化
5. 第五步：实现柱状图、散点、误差线与显著性标注
6. 第六步：实现整版拼图与 SVG/PDF/CSV 导出
7. 第七步：增加项目保存、参数回放、分析审计信息

## 建议的产品定位
这不是“另一个 ImageJ”，而是：
- 面向 WB/Dot Blot/凝胶图的分析工作流工具
- 强调 publication workflow，而非通用图像处理
- 把“算灰度 + 统计 + 排版出图”做成一个完整闭环

如果定位清晰，这个方向是成立的，而且比单纯做“ImageJ 的 Web 版”更有价值。
